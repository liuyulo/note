Memory Model

always create new places in memory when assigning

variable assign 
a = 5
# a: id1    id1: 5
a = 3
# a: id2    id2: 3
b = a
# a: id2    id2: 3
# b: id2
a = 5
# a: id1    id1: 5
b = a + 2
# a: id1    id1: 5
# b: id2    id2: 7
a = 10
# a: id3    id3: 10
# b: id2    id2: 7
list 
l = [12, 23]
# l: id1    id2: 12
#           id3: 23
#           id1: { 0: id2, 1: id3 }
l[1] = 34
# l: id1    id2: 12
#           id1: { 0: id2, 1: id4 }
#           id4: 34
l += [56] # create new list
# l: id6    id2: 12
#           id4: 34
#           id7: 56
#           id6: { 0: id2, 1: id4 }
a = l # copy reference aka alias
# l: id6    id2: 12
# a: id6    id4: 34
#           id7: 56
#           id6: { 0: id2, 1: id4 }
b = l + [78] # copy by value
# l: id6    id2: 12
# b: id9    id4: 34
#           id7: 56
#           id8: { 0: id2, 1: id4 }
#           id9: { 0: id2, 1: id4, 2: id10 }
#           id10: 78

String

string is immutable - str.find(sub[, start[, end]]) - return the lowest index in the string where substring sub is found within the slice s[start:end] - start and end: in slice notation Return -1 if sub is not found. - should only be used to get the position

List

  • slicing always return list
  • b = a copies reference
  • list passes reference to functions
  • use List[str] or list as type hint in functions

Methods

  • list.append(x)
    • aka vector::push_back(x)
  • list.extend(iterable)
    • aka a += iterable
  • list.insert(i, x)
    • aka a = a[:i] + [x] + a[i:] (x become i-th element)
  • list.remove(x)
    • remove first item of value x from the list, or ValueError
  • list.pop([i])
    • remove the last or i-th item and return it
  • list.sort()
    • sort self
